## Виджет обратной связи (фича‑реквесты и баг‑репорты)

### Цель
Добавить на каждую страницу плавающий виджет в правом нижнем углу, который открывает форму для отправки фича‑реквестов и сообщений об ошибках автору (через email/Resend или другие каналы), без привязки к конкретной внешней инфраструктуре, чтобы при необходимости можно было легко доработать или отключить.

### Архитектура

- **Frontend (React)**
  - Общий компонент `FeedbackWidget`:
    - Плавающая кнопка в правом нижнем углу (иконка + подпись).
    - Модальное окно с формой.
  - Подключение `FeedbackWidget` на уровне `App.jsx`, чтобы он был доступен на всех страницах.
  - Поля формы:
    - Тип: `Ошибка` / `Фича‑реквест` (radio).
    - Тема (строка).
    - Описание (textarea).
    - Контакт (обязательное поле: email или `@username`).
    - Автоматически подставляемые поля (readonly/hidden):
      - `page_url` = `window.location.href`.
      - Опционально: `user_agent` = `navigator.userAgent`.
  - UX:
    - Кнопки: «Отправить» / «Отмена».
    - После успешной отправки — уведомление («Спасибо, ваш запрос отправлен»).
    - Обработка ошибок отправки с понятным сообщением пользователю.

- **Backend (FastAPI) — опционально**
  - Pydantic‑модель `FeedbackRequest`:
    - `type` (`bug` / `feature`),
    - `title`,
    - `description`,
    - `contact`,
    - `page_url`,
    - `user_agent` (опционально).
  - Эндпоинт `POST /api/feedback`:
    - Валидирует входные данные.
    - Логирует в стандартный лог (для локальной разработки этого достаточно).
    - Опционально:
      - сохраняет в таблицу `feedback` в БД,
      - отправляет email через Resend (если разработчик настроил ключи).
  - Конфигурация:
    - Использовать флаги/переменные окружения, чтобы:
      - включать/выключать фактическую отправку почты,
      - управлять адресом получателя (email автора).

### Поведение по умолчанию (для локального развёртывания)

- Виджет **включён**, но:
  - Если Resend / почта не настроены:
    - сервер либо:
      - только логирует запросы в лог,
      - либо сохраняет в БД без отправки писем.
- Документация в `README.md`:
  - Кратко описать:
    - как включить/отключить виджет,
    - как настроить Resend (или другой канал),
    - где смотреть локально собранный фидбэк (логи/таблица `feedback`).

### Возможные дальнейшие улучшения

- Добавить переключатель/настройку в `.env`:
  - `FEEDBACK_ENABLED=true/false`,
  - `FEEDBACK_SEND_EMAIL=true/false`.
- Добавить интеграцию с Telegram‑ботом:
  - бекенд по желанию дублирует фидбэк в Telegram‑канал/чат.
- Добавить прикрепление текущего `transcript_id` или `rag_session_id`, если виджет открыт на соответствующей странице.
